  3. Fluxo de Dados e Raciocínio (LLM Cloud)
  Como operamos com modelos grandes em nuvem (Gemini 2.5), o fluxo de dados foi configurado para Zero Perda:
   1. Imagem: Enviada em alta definição (até 1024px).
   2. Sinal: O vetor numérico completo (5000 pontos por derivação) é injetado no prompt.
   3. Contexto: Janela de 8192 tokens garantida para que a LLM "leia" o ECG digitalizado antes de concluir.

  Conclusão do Relatório
  O produto possui uma inteligência clínica instalada (Hubert, Queenbee, ECGFounder e Rx) que já supera qualquer modelo visual genérico. No entanto, o fundamento da digitalização (os arquivos vazios de 0 bytes) está impedindo que os
  modelos de sinal recebam dados.


  O que falta no produto para ser 100% funcional:
   1. Materializar os arquivos `ecg_segmentation_unet.pt` e `resnet_code15.h5` (atualmente vazios no seu PC).
   2. Finalizar a configuração de caminhos do nnU-Net assim que os pesos forem localizados.
